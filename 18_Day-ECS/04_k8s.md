# Kubernetes Architecture (Structure)

## 🔹 Overview
Kubernetes (K8s) is a **container orchestration system** that follows a **master-worker (control plane + nodes)** architecture.

---

## 🔥 1. Control Plane (Master Node)

The control plane is the **brain of Kubernetes**. It decides what happens in the cluster.

### Components:

### 1. API Server
- Entry point of Kubernetes
- Accepts commands (kubectl, UI, API calls)
- Validates and processes requests

---

### 2. Scheduler
- Assigns pods to worker nodes
- Decides *where* a container should run

---

### 3. Controller Manager
- Ensures cluster state matches desired state
- Example: If a pod crashes, it restarts it

---

### 4. etcd
- Distributed key-value database
- Stores all cluster data (config, state, metadata)

---

## 🖥️ 2. Worker Nodes

Worker nodes actually **run the applications (containers)**.

### Components:

### 1. Kubelet
- Agent running on each node
- Communicates with control plane
- Ensures containers are running as expected

---

### 2. Container Runtime
- Runs containers (e.g., Docker, containerd)
- Responsible for starting/stopping containers

---

### 3. Kube Proxy
- Manages networking inside the node
- Handles communication between services

---

## 📦 3. Pods

- Smallest deployable unit in Kubernetes
- Contains one or more containers
- Shares network + storage

👉 Example:
- A web app container + logging container can be in one pod

---

## 🔹 Simple Summary

- Control Plane = Brain (decides everything)
- Worker Nodes = Muscles (run containers)
- Pods = Smallest unit (where apps live)
- etcd = Memory (stores all data)
