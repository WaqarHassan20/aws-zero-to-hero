### Lambda Functions
- Lambda functions are anonymous functions that can be defined in a single line of code. They are often used for short, simple operations that can be expressed in a concise way.
- They are event driven, meaning they can be triggered by various events such as HTTP requests, database changes, or file uploads.
- Lambda functions are stateless, meaning they do not maintain any state between invocations. Each time a lambda function is invoked, it starts with a clean slate. And after the execution, any state or data created during the execution is discarded.
- They are typically used in serverless architectures, where you can run code without provisioning or managing servers. This allows for greater scalability and cost efficiency, as you only pay for the compute time you consume.

### Difference between Server and Serverless
- In a traditional server-based architecture, you have to manage and maintain the server infrastructure, including hardware, operating system, and software. You are responsible for scaling the server resources based on the demand.
- In a serverless architecture, you only pay for the compute time you consume. You do not have to manage the server infrastructure, and the server is automatically scaled based on the demand.
- For EC2 you have to manually set up auto-scaling and then terminate them, while in Lambda, the scaling is handled automatically by the cloud provider.
- Are they related to state or stateless? No, both server-based and serverless architectures can be either stateful or stateless, depending on how the application is designed. The key difference lies in how the infrastructure is managed and scaled, rather than the state management of the application itself.
- What is Stateful and Stateless? Stateful applications maintain state between requests, while stateless applications do not.
- In a stateful application, the server retains information about the client's session, allowing for a more personalized experience. In contrast, a stateless application treats each request as independent, without any knowledge of previous interactions.
- Stateful = “I remember you” - Stateless = “Who are you every time?”

### Example of lambda function using AWS volumes service:
- Let's suppose you are bound not to use the GP2 volume type, and you have to use the GP3 volume type.
- You can create a lambda function that generates the notification to dev team and after that lambda function terminates the volume.
- the key is that it will only run when the event is triggered, and you will not be charged for idle time.

### Example of lambda function using AWS S3 Bucket service:
- You are bound to not use the bucket with public access, and you have to use the bucket with private access.
- You can create a lambda function that generates the notification to dev team and after that lambda function changes the bucket access to private.
- the key is that it will only run when the event is triggered, and you will not be charged for idle time.

| Feature              | Server-Based Architecture                | Serverless Architecture                                   |
| -------------------- | ---------------------------------------- | --------------------------------------------------------- |
| **Infrastructure**   | You manage servers (VMs like Amazon EC2) | No server management (handled by cloud, e.g., AWS Lambda) |
| **Setup Complexity** | High (OS, scaling, networking)           | Low (just deploy code)                                    |
| **Scaling**          | Manual or auto-scaling setup required    | Automatic scaling out-of-the-box                          |
| **Cost Model**       | Pay for uptime (even idle)               | Pay per execution (only when code runs)                   |
| **Performance**      | No cold starts (always running)          | Cold start delay possible                                 |
| **Control**          | Full control over server                 | Limited control                                           |
| **Maintenance**      | You handle updates, patches              | Managed by provider                                       |
| **Deployment**       | Deploy full app on server                | Deploy small functions (microservices style)              |
| **Use Case**         | Long-running apps, custom setups         | Event-driven apps, APIs, automation                       |
| **Examples**         | Node.js app on EC2, VPS hosting          | APIs via Lambda, Firebase functions                       |


### What is corn job?

- A cron job is a way to schedule tasks to run automatically at specific times or intervals on a Unix/Linux system.

- 👉 “Run this command automatically later or repeatedly”

- It uses a service called cron, which reads a configuration file called crontab.
